## 已复核的关键行为

### 统一数据流

- 应用只挂载 `DataRegistryProvider`。
- `useAnimeList()` 只读取注册表，不再重复请求 YUC。
- AGE 数据进入动漫档案、搜索、详情、收藏、资源、写作和审查流程。

### 数据正确性

- 缺失字段保持空值。
- 占位文字、空数组和模板变量被识别为无效值。
- 新抓取的空值不会擦除旧有效字段。
- AGE 缺失年份不写入当前年份。
- YUC／AGE 年份冲突不强制合并。
- 资源保留作品 ID、类型、线路、集数、来源、授权状态、可用性和抓取时间。

### 采集架构

- `/api/age/current` 一次只读取指定分类的一页。
- 完整采集由 `sync-age.mjs` 负责，保存可恢复快照。
- Worker 可优先读取内置快照。
- 分类页、首页、周更、专题和热榜独立失败，不再强制让所有数据一起失败。
- 同步器并发写入已串行化，避免临时文件竞争。

### 前台

- 资源中心不再固定截断到前 36 条。
- AGE 条目详情可以按来源 ID读取真实详情。
- 写作中心根据注册表字段生成草稿，并显示缺失项、冲突和来源。
- 数据审查不再只看三个字段。
- 详情页隐藏无值字段，不显示大量“未公开”。
