# 次元生成局统一数据与资源功能实施报告

实施日期：2026-07-23

## 一、实施目标

依据上一轮代码审查，修复以下阻断问题：

1. YUC 与 AGE 使用两套 Provider，主站无法使用 AGE 数据。
2. AGE 条目详情无法打开，资源中心只显示前 36 条 YUC 数据。
3. AGE 一次前台请求实时抓取约 181 个分类页面。
4. 写作中心和数据审查页只是页面外壳。
5. YUC 解析器制造“待补全”、通用简介和固定平台等错误数据。
6. 媒体地址被主动删除，资源类型与来源状态不足。
7. 解析器依赖 HTML 属性顺序，异常 URL 或编码可能中断解析。
8. 缺失年份被写成当前年份，空数据可能覆盖旧数据。

## 二、完成内容

### 1. 统一数据层

- `DataRegistryProvider` 成为唯一网络数据入口。
- `useAnimeList()` 改为注册表兼容入口，不再发起第二套请求。
- 首页、搜索、档案、详情、季度、日历、资源、写作和审查读取同一注册表。
- AGE 条目进入统一搜索、收藏、档案和 `/anime/:id` 详情路由。

### 2. 安全合并与来源

- 增加字段来源、资源来源、热度口径、授权状态和可用性类型。
- 空值、空数组、模板变量和占位值不会覆盖旧有效字段。
- YUC 与 AGE 仅在规范化标题一致且年份不冲突时合并。
- 年份冲突保留两个条目并生成冲突说明。
- AGE 年份缺失时保持未知，不使用当前年份代填。

### 3. YUC 解析

新增独立 `scripts/yuc-parser.mjs`：

- 解析星期、时间、首播日期和平台。
- 解析页面实际提供的原名、原作类型、题材、制作公司、导演、系列构成、人物设计、音乐、CAST、官网和 PV。
- 字段缺失时保持空值。
- 不生成通用简介、通用推荐、虚假 STAFF 或“平台请以官方公告为准”字段。

### 4. AGE 解析与资源

重写 `scripts/age-parser.mjs`：

- 作品卡解析不依赖 HTML 属性顺序。
- 支持首页、热榜、分类、周更、新番专题、详情和播放页。
- 保留详情页、线路、分集和媒体 URL。
- 识别站点级镜像与防走丢入口。
- 单条非法 URL 和异常百分号编码不会中断整页。
- 媒体资源默认标记为授权未知、可用性未检查，不冒充官方资源。

### 5. AGE 同步架构

新增 `scripts/sync-age.mjs`：

- 支持日漫、国漫、动态漫、剧场、特摄和美漫。
- 按分页同步并保存 `public/data/age-latest.json`。
- 支持断点续传、超时、重试、限速、并发限制、失败清单和详情／播放可选同步。
- 快照写入使用串行队列和原子重命名，避免并发页面写入损坏 JSON。
- Worker 前台接口只读取一个分类页；不再用一次请求实时抓取 181 页。
- 已存在快照时优先读取最近成功版本。

### 6. 前台功能

- 资源中心支持关键词、来源、资源类型、AGE 分类、页码和分页筛选。
- YUC 外部链接转换为统一资源记录。
- AGE 条目详情页按需读取详情、线路和播放页媒体。
- 写作中心从注册表选择作品，显示缺失项、冲突和字段来源，并仅用真实字段生成草稿。
- 数据审查页检查身份、来源、日期、集数、STAFF／CAST 占位、外部链接、资源 URL、作品绑定、来源 ID、字段覆盖和全局冲突。
- 详情页仅显示有值的 STAFF、CAST 和资料字段，不再固定展示“未公开”宫格。

使用临时模块声明执行严格 TypeScript 内部检查，未发现项目源码内部类型错误。

## 四、未完成和客观限制

- 当前环境 npm 镜像返回 503，无法安装 React、Vite 等真实依赖，因此未能独立完成真实 `npm run typecheck` 和 `npm run build`。
- 成品包未预置 AGE 全站快照。同步器和快照读取逻辑已经实现，但完整同步应在网络正常、允许访问上游且设置合理限速的环境运行。
- 外部网站可能修改页面模板或提供错误数据；系统通过来源、抓取时间、状态、测试和冲突报告降低风险，但不能保证外部原始内容绝对无误。
- 跨来源自动合并采用保守的标题与年份规则，复杂别名仍需人工审查。
