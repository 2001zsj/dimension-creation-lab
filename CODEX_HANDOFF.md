# Codex 交接说明

## 已完成修改

1. 修复日本时间/本地时间切换：现在会同时换算星期和时刻，并处理 `24:xx` 跨日表记。
2. 消除公开资料冒充个人记录的问题：公开同步条目不再自动设置“在追”、观看进度、评分或观看日志。
3. 移除 `progress + 1` 推断下一集的错误逻辑，改为展示首播日期、本周放送或明确的个人进度。
4. 首页当前季度、季度入口、统计卡和来源说明改为根据数据动态计算。
5. 新番雷达改为根据真实数据动态生成全部季度分组，不再只显示四个写死分组。
6. 首页“雷达目标”统计使用完整结果，不再先截取 4 条后统计。
7. “角色生成器”改名为“角色设定组合器”，明确这是本地随机组合，不冒充真实 AI 模型。
8. 修复文章和提示词在同一路由内切换 hash 时不滚动的问题。
9. 增加路由滚动恢复：新页面回到顶部，浏览器后退恢复原列表位置，hash 跳转定位对应内容。
10. 改善导航菜单、搜索弹窗、Tab、筛选按钮、开关和图片替代文字的可访问性。
11. 图片加载失败时回退到渐变占位图；修复 `.cover-image` 被通用 CSS 规则覆盖定位的问题。
12. Worker 新增统一接口 `/api/anime/current`，资料期通过 `YUC_PERIOD=YYYYMM` 配置。

## 需要 Codex 继续执行

1. 在网络正常的环境运行：
   ```bash
   npm ci
   npm run typecheck
   npm run build
   ```
2. 启动本地站点并使用 Playwright 检查以下尺寸：1440×900、1280×800、768×1024、430×932、390×844、375×812。
3. 重点回归：
   - 日历日本时间与本地时间切换后，条目星期和时间是否正确；
   - 搜索结果从文章/提示词之间连续跳转时是否定位正确；
   - 从动漫长列表进入详情后返回，滚动位置是否恢复；
   - 移动端菜单、搜索弹窗焦点锁定和 Esc 关闭；
   - 远程封面加载失败时是否显示渐变占位；
   - 公开资料详情页不应出现“我的记录”“在追”“观看日志”或“REAL DATA”。
4. 使用 axe-core 检查颜色对比度、表单标签、按钮可访问名称和 Tab 语义。
